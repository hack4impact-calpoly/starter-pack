Lab 34 Colors and Text

Goals

  • Learn about colors and text in CSS

Color

Some CSS properties take a color as its value. CSS comes with a few keywords which define built-in colors. You can use red, aqua, antiquewhite, blue, silver, transparent, and more to reference colors.

CSS also accepts RGB, HSL, and hexadecimal colors. If you've ever used a digital art program or Photoshop you probably know what these are, but if you haven't, here's a quick rundown.

  • RGB stands for red, green, blue and describes the amount of each color used in combination to make other colors. Defined in CSS as rgb(0, 0, 255) with each number between 0 and 255.

    • Note: You can also use RGBa as rgba(0, 0, 255, 0.5)to define an alpha or transparency value between 0 and 1.

  • HSL stands for hue, saturation, lightness and describes the color in terms of these values. Defined in CSS as hsl(240, 100%, 50%) with the first number being the hue angle on a color wheel between 0 and 360, the second being the saturation between 0% and 100%, and the third being the lightness between 0% and 100%.

    • Note: You can also use HSLa as hsla(240, 100%, 50%, 0.5).

  • Hexadecimal colors are a way of representing RGB colors in a more compact way. Defined in CSS as #0000ff with the first two characters representing the amount of red, the second two characters representing the amount of green, and the last two characters representing the amount of blue. Each character is a hexadecimal (base-16) digit between 0 and f.

    • Note: If the pairs share a letter it can be shortened even further to three characters. #0000ff can be shortened to #00f.

Text

The different text properties that we will consider are font-size, font-family, font-weight, font-style, text-decoration, text-transform, and text-align.

Some of these are pretty self-explanatory, but we'll go over them!

  • font-size changes the size of the font. Common units are px or pixels and pt or point like in word processing programs.

    • Note: Make sure you don't use font-size as a crutch instead of proper semantic HTML e.g. headings. Headings should use the proper tags and not be large font paragraphs.

  • font-family affects the actual font itself. In order to load fonts, browsers will need the font to be on a website visitor's computer. There are a few web-safe fonts that are generally available across most browsers and systems. You can define multiple comma-separated fonts as a backup in case a font is not found.
    • font-family: "Helvetica", "Arial", "sans-serif";

    Note: If you want to use a font that's not a standard font, we highly recommend using Google's Web Fonts. Their tutorial on how to import them is here: https://developers.google.com/fonts/docs/css2

  • font-weight determines whether a font is bold or normal. You can also specify weights in numbers e.g. (100, 200, 500) but we suggest sticking with bold or normal.
  • font-style determines whether a text is italic or not.
  • text-decoration describes lines above, through, or below text. It can take the following values: underline, overline, line-through and none.
  • text-transform will change the case of the text. It can take the following values: capitalize, lowercase, uppercase, and none.
  • text-align will align text within it's parent container. It can take the following values: left, right, center, and justify.
