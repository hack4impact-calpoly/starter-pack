Lab 12 Staging Changes

Goals

  • Learn how to stage changes for later commits

Add Changes to Staging Area

Use the following command to move the modified files to the staging area! Then check the status again.

Execute

git add README.md
git status

Output

git status
(out)On branch main
(out)Changes to be committed:
(out) (use "git restore --staged ..." to unstage)
(out) modified: README.md
(out)

Similar to tracking a file, you've staged the file to the staging area. Git now knows about your changes, but it's not permanently saved to the Git repository yet. The next commit will include only the staged changes.

If you decide you don't want to commit that change after all, the status command reminds you that the git restore command can be used to unstage that change.
