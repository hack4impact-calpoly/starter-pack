Lab 10 Tracking Files

Goals

  • Learn how to track a file to the Git repository

Add File to Track

Thankfully, there's a helpful suggestion about what we can do! To begin tracking this file in our Git repository, run the following command:

Execute

git add index.html

Now run, git status again to check the state of the working directory and repository.

Execute

git status

Output

git init
(out)On branch main
(out)
(out)No commits yet
(out)
(out)Changes to be committed:
(out) (use "git rm --cached <file>..." to unstage)
(out)        new file:   README.md
(out)

When you create new files, you should track them as soon as possible.

If you remember Lab 7, the file is now added to the staging area. From there, the next step is...

Commit File to Repository

To save the state of this file, we'll commit it to our Git repository! Every commit also requires a message that describes the purpose of the commit, which we can set with the -m parameter.

Execute

git commit -m "Initial commit"

Output

git commit -m "feat: add README.md"
(out)[main (root-commit) f1et44a7] feat: add README.md
(out) 1 file changed, 0 insertions(+), 0 deletions(-)
(out) create mode 100644 README.md
(out)

We will be using conventional commits. The format is <feat or fix>: <commit message under 50 words>. Commit messages should be clear and concise in the present tense.

Status Check

Execute

git status

Output

git status
(out)On branch main
(out)nothing to commit, working tree clean
(out)

You should see this message which means the state of your working directory matches the latest state committed to the repository!

More Files to Track

Create the following files, follow the same steps, and track them one at a time. For each file, the commit message should follow the format of "feat: add <file name>".

  • styles.css
  • blog.html
  • contact.html
  • index.html
  • portfolio.html
  • resume.html

Alternatively, you can make one commit with all the files. In which case, the commit message should be "feat: <describe commit in 50 words>".

Hint: You can stage more than one file before you commit them.
