Cascading

If you have two style rules with the same selector, then the selector that appears last will have its style applied. For example, trying to select paragraph elements twice with two styles like so: 


      
The second HTML paragraph selector is last so it will take precedence. The text color will be blue.

      
The More Specific, the Better

      
Okay, not really... but in CSS when there are conflicting style rules, the winner
        is decided by specificity. You are unlikely to have a situation where you select the same
        element twice,
        however conflicts with nested elements are more likely to occur:

      

      
Since the rule for the <p> blue text color comes last, it may seem as
        though
        that will be the rule that is applied, however that is not the case. The first rule is more
        specific so it is given more importance.
      

      
Calculating Specificity

      
A quick and easy way to calculate specificity is to assign different selectors, different values.
        Give every ID selector a value of 100, every class selector
        a value of 10, and every element selector a value of 1.
      

      
    
        
  • p would have a value of 1
    • 
        
  • div p would have a value of 2
    • 
        
  • .class would have a value of 10
    • 
        
  • div p .class would have a value of 12
    • 
        
  • #id would have a value of 100
        
    • 
        
  • div #id .class p would have a value of 112
    • 
      

      
In practical use, div p would rule over p
        and div #id .class p would rule over us all.
