CSS Selectors

If HTML has tags, CSS has selectors. Every selector has a pair of curly brackets associated with it. Inside the curly brackets, are properties with a colon assigning them a value. Every property is separated from another by a semi-colon. 



      
It is difficult to cover all of the tags. If you're ever uncertain
        if a property applies to a tag, the best choice is to just try it. The nice thing about CSS being all about
        appearance is that you can apply a property and refresh your webpage to see if it works!
      

      
Worse comes to worst, you can use the references here: https://developer.mozilla.org/en-US/docs/Web/CSS/Reference
        to browse the entire list of every single CSS property and more.

      
Note: As always, during this Starter Pack, GitHub Discussions is open for you
        to ask for help!

Class and ID Selectors

      
When we were coding our HTML, we added classes="" as attributes to various HTML tags.
        The reason we did this was so that we could group specific elements together. Now when we are coding our
        CSS, we can apply styles specifically to elements with specific classses.

      
Class selectors are a period followed by the class name. ID selectors are a hashmark or pound symbol
        followed by the ID name.

      

      
Note: Remember that classes group multiple tags together while IDs can only be
